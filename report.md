# Business use case:
I chose models from Gemini and OpenAI for testing and found that the results were identical. As my test set was relatively simple and included certain constraints—such as rules and response formats—I ultimately found that both Gemini and OpenAI’s ChatGPT performed the tasks exceptionally well.
# baseline vs. final prompt
The prompts for the initial model were relatively simple, so the output was a paragraph that did indeed summarise the meeting’s content and filtered out some irrelevant noise, but it still required manual review. I then added rules to encourage simpler and more structured responses; whilst this resulted in a more concise output and left out certain details—such as the name of the person in charge—the format remained a paragraph. Finally, I added a response format to restrict the output, and the result was a task list in JSON format.
# flaw and suggestions for deployment
The current version is capable of extracting tasks from meetings, but manual review is still required; users must manually identify and add their own tasks from the JSON format. This aspect requires further improvement; for example, it could be enhanced to allow filtering by name.
My current recommendation is to deploy this workflow, subject to several prerequisites: 1. An additional speech-to-text function must be deployed. 2. Following deployment, consent must be obtained from all participants to record the meeting. 3. It is recommended that the workflow be deployed on-premises or using the company’s own model, as sensitive meeting content should not be exposed to the internet.
